# 解码方法

- **难度**: 中等
- **分类**: 一维动态规划
- **考点**: 动态规划, 字符串
- **链接**: [NeetCode](https://neetcode.io/problems/decode-ways) | [力扣 91](https://leetcode.cn/problems/decode-ways/)

## 题目描述

一条包含字母 A-Z 的消息通过以下映射进行了编码：'A' -> "1"，'B' -> "2"，...，'Z' -> "26"。给你一个只含数字的非空字符串 `s`，请计算并返回解码方法的总数。题目数据保证答案肯定是一个 32 位整数。注意，像 "06" 这样的分组是无效的，因为 "06" 不是有效的编码（只有 "6" 是有效的）。以零开头的字符串没有有效的解码方式。

## 示例

**示例 1:**

```
输入: s = "12"
输出: 2
解释: "12" 可以解码为 "AB"（1 2）或者 "L"（12）。
```

**示例 2:**

```
输入: s = "226"
输出: 3
解释: "226" 可以解码为 "BZ"（2 26）、"VF"（22 6）或者 "BBF"（2 2 6）。
```

**示例 3:**

```
输入: s = "06"
输出: 0
解释: "06" 无法解码。前导零使编码无效。
```

## 约束条件

- `1 <= s.length <= 100`
- `s` 只包含数字，可能包含前导零。

## 函数签名

```go
func numDecodings(s string) int
```
