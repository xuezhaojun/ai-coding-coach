# 网络延迟时间

- **难度**: 中等
- **分类**: 高级图
- **考点**: Dijkstra 算法, 堆, 最短路径, 加权图
- **链接**: [NeetCode](https://neetcode.io/problems/network-delay-time) | [力扣 743](https://leetcode.cn/problems/network-delay-time/)

## 题目描述

有 `n` 个网络节点，标记为 `1` 到 `n`。给你一个列表 `times`，表示信号经过有向边的传递时间。`times[i] = (ui, vi, wi)`，其中 `ui` 是源节点，`vi` 是目标节点，`wi` 是一个信号从源节点传递到目标节点的时间。

现在，从某个节点 `k` 发出一个信号。需要多久才能使所有节点都收到信号？如果不能使所有节点收到信号，返回 `-1`。

## 示例

**示例 1:**

```
输入: times = [[2,1,1],[2,3,1],[3,4,1]], n = 4, k = 2
输出: 2
解释: 从节点 2 发出的信号，1 个时间单位到达节点 1，1 个时间单位到达节点 3，2 个时间单位到达节点 4。
```

**示例 2:**

```
输入: times = [[1,2,1]], n = 2, k = 2
输出: -1
解释: 从节点 2 无法到达节点 1。
```

**示例 3:**

```
输入: times = [], n = 1, k = 1
输出: 0
解释: 只有一个节点，信号已在该节点。
```

## 约束条件

- `1 <= k <= n <= 100`
- `1 <= times.length <= 6000`
- `times[i].length == 3`
- `1 <= ui, vi <= n`
- `ui != vi`
- `0 <= wi <= 100`
- 所有 `(ui, vi)` 对互不相同（没有重边）。

## 函数签名

```go
func networkDelayTime(times [][]int, n int, k int) int
```
